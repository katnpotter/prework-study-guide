HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An element is everything between an opening and closing tag.
  • A tag is contained inside angle brackets.
  • Common elements used are: text, image, header, and list elements.
  • Semantic elements are an example of body elements and they are used to describe its content and its role.
  • Semantic elements make the website more accessible and the HTML easier to read.
  • An attribute is additional information about an element.
  • The doctype declaration tells the browser what type of document to expect when opened.
  • Declarations are not case sensitive but elements are.
  • Headings should be used in order of size and the largest heading should only appear once.
  • The body element is known as a parent element and the ones on the inside are called child elements.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Three ways to style a webpage are with inline CSS, and internal and external style sheets.
  • Inline CSS is a way to style an element directly, or independently.
  • Internal style sheets are contained inside the head section and are applied to multiple elements.
  • External style sheets are all CSS rules of a document kept in a linked file in the head section.
  • Declarations contain the property and the value of the property.
  • The Do Not Repeat Yourself (DRY) code is a coding principle that encourages you to reuse or share pieces of code reduce the amount that needs to be rewritten.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git push origin branch name: pushes all changes of that branch to the remote repository
  • git pull origin main: pulls all changes from the remote branch to the local branch
  • git commit -m "short description": commits all changes to the remote repository
  • git add -A: prepares to add any changes before commits
  • pwd: tells us what folder we're currently in
  • cd ..: moves back a folder in the directory
  • cd 'folder name': takes us to that folder on our harddrive
  • touch 'file name': creates that file in the folder you're currently in

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A semicolon at the end of a line is where a statement ends.
  • JavaScript is case sensitive.
  • Boolean is a true or false value. They are special keywords that do not need quotations.
  • JavaScript is a weakly typed language.
  • Strings are all surrounded by quotation marks.
  • In JavaScript, code runs linear from top to bottom.
  • Conditional statements and loops control the order that code is executed or if it is at all.
  • An if statement is a way of introducing decision making into code.
  • If statements only execute if the statement meets a condition or if it is true.
  • When a computer encounters an if statement it will check to see if it is truthy or falsy.
  • Falsy values include null, undefined, 0, and NaN (Not a Number).
  • An array is a single variable that is used to hold a group of data.
  • The first item in any array has an index of 0, not 1.
  • A 'for' loop uses the pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Commenting out code makes it inactive without deleting it.
  • JavaScript is an object-oriented programming (OOP) language.
  • A function is a set of instructions that tells the computer how to perform a certain task.
  • Functions have to be called in order to be executed.
  • A function's name is followed by parenthesis.
